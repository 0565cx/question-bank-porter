# 搬运工（Question Bank Porter）

把审题 skill 的产出对接到「各岗位AI审题进度」仓库。使用者每轮审核完（一二轮视为一轮）、重新出完题、最后汇总完、或做完语言问题审核后，调用本 skill 即可自动精准上传到对应岗位文件夹。

## 仓库与目录约定

目标仓库本地路径（默认）：

```
/Users/risei/Desktop/北森-测评产品经理/AI面试官/专业能力/ai-question-review-progress
```

远程：`https://github.com/0565cx/ai-question-review-progress`（private）。

每个岗位下的目标文件夹：

| 文件类型 | 目标文件夹 |
|---|---|
| 保留题目表 | `2_保留题目/` |
| 需重新出题表 | `3_需重新出题/` |
| 重新出答案表 | `4_需重新出答案/` |
| HTML 审核看板 | `5_审核结果看板/` |
| 收尾题目（知识审核最后一轮重新出的题） | `6_收尾题目/` |
| 最终汇总保留题目 | `7_汇总/` |
| 待审核题目 | `1_待审核题目/` |
| 待语言问题审核 | `8_语言问题审核/1_待语言问题审核/` |
| 语言问题一审结果 | `8_语言问题审核/2_一审结果/` |
| 语言问题待重出 | `8_语言问题审核/3_待重出/` |
| 语言问题重出结果 | `8_语言问题审核/4_重出结果/` |

岗位路径以仓库 `data/positions.csv` 为准（按 `子行业` + `岗位名称` 匹配）。

## 使用前确认

调用本 skill 时，向用户确认：

1. 源文件或目录路径（审题 skill 的产出位置）。可以是某轮的「X轮审核结果汇总」目录、「X轮审核后需要重出的内容」目录、待审核题目目录、或语言问题审核产出目录。
2. 本次是哪种搬运：每轮审核结果（含 3 个表 + 看板）、待审核题目、还是语言问题审核表。
3. 仓库本地路径（若非默认路径）。

不要向用户询问行业/岗位/轮次，这些从文件名或父目录路径自动解析。

## 搬运工作流

复制此清单并逐步跟踪：

```
搬运进度：
- [ ] 步骤1：确认源路径与仓库路径
- [ ] 步骤2：预演（dry-run）输出搬运计划
- [ ] 步骤3：检查无法识别项，必要时改名/补全
- [ ] 步骤4：把搬运计划给用户确认
- [ ] 步骤5：执行搬运（--apply）
- [ ] 步骤6：git 提交并推送
- [ ] 步骤7：确认看板自动更新
```

### 步骤2-3：预演与改名

先 `git pull` 同步仓库，再预演。脚本会解析每个文件的 `子行业 / 岗位 / 文件类型 / 轮次`，规划目标路径，并对不规范文件名给出规范化后的新名字。

```shell
cd <仓库路径>
git pull --no-rebase
python3 ~/.cursor/skills/question-bank-porter/scripts/port.py \
  --repo <仓库路径> \
  --src "<源目录或文件1>" \
  --src "<源目录或文件2>"
```

输出包含「搬运计划」和「⚠️ 无法识别」两部分。

- 对「⚠️ 无法识别」的文件：根据上下文判断它属于哪个岗位/类型，可重命名源文件后重跑预演，或直接告诉用户哪几个文件无法自动归位、需要人工确认。
- 不要把无法识别的文件随意丢进某个文件夹。

每轮（一二轮视为一轮）审核完的典型来源：`X轮审核结果汇总` 或 `X轮审核后需要重出的内容` 目录里的三个表（`*可保留的题目*`、`*需要重新出题*`、`*重新出答案*`）+ `roundX_dashboard.html`。直接把这两个目录一起作为 `--src` 传入即可，看板的岗位会从父目录路径推断。

### 步骤4-5：确认后执行

把「搬运计划」清单（每个文件 → 目标路径、是否改名）展示给用户。用户确认后，加 `--apply` 执行复制：

```shell
python3 ~/.cursor/skills/question-bank-porter/scripts/port.py \
  --repo <仓库路径> --src "<源…>" --apply
```

### 步骤6-7：提交推送

```shell
cd <仓库路径>
git add -A
git commit -m "📦 搬运 <行业-岗位> 第X轮审核结果"
git push
```

推送后 GitHub Actions 会自动重新生成 `进度看板.md` 和 `index.html`。提醒用户稍等片刻看板即会更新。

## 文件名解析规则

脚本按以下关键词识别文件类型（详见 `scripts/port.py`）：

- 含「语言」→ 语言问题审核四个子文件夹之一（按「待语言问题审核 / 一审结果 / 待重出 / 重出结果」细分）。
- 含「收尾」→ `6_收尾题目`。
- 含「dashboard」或「看板」→ `5_审核结果看板`。
- 含「汇总 / 最终保留 / 审核通过」→ `7_汇总`。
- 含「可保留 / 保留题目」→ `2_保留题目`。
- 含「重新出答案 / 重出答案」→ `4_需重新出答案`。
- 含「需要重新出题 / 重新出题」→ `3_需重新出题`。
- 含「待X轮审核」→ `1_待审核题目`。

轮次从「待X轮审核」「X轮_」「第X轮」中解析，支持中文数字。子行业、岗位先从文件名取，取不到再从父目录路径取。

## 改名规范

改名后统一格式：

- 待审核题目：`<子行业>-<岗位>题目_待X轮审核.xlsx`
- 保留/重出题/重出答案：`X轮_<子行业>_<岗位>_<可保留的题目|需要重新出题|重新出答案>.xlsx`
- 看板：`第X轮审核看板.html`
- 收尾题目：`<子行业>-<岗位>_收尾题目.xlsx`
- 汇总：`<子行业>-<岗位>_最终汇总保留题目.xlsx`
- 语言审核：`<子行业>-<岗位>_<待语言问题审核|语言问题一审结果|语言问题待重出|语言问题重出结果>.xlsx`

## 注意事项

- 搬运前必须 `git pull`，避免和 Actions 机器人提交冲突。若 push 冲突，`git pull --no-rebase` 后重推。
- 仓库为 private，含敏感题目，不要把内容外发。
- `--apply` 前一定先给用户看搬运计划。
